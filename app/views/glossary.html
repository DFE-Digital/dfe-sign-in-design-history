A to Z Glossary

A collection of terms commonly used across the DfE Sign-In service

From: DfE Sign-In team

Published: 27th September 2023

Last updated: 27th September 2023

A

Approver

An elevated permission level for users in DfE Sign-in.

B

BAU team

The BAU team provides support for the DfE Sign-in service for both users and services.

C

Client ID

The client ID is a unique identifier of the service users want to configure. It is created manually by the DfE Sign-in team.

Client secret

The client secret is a value that acts as a password for the service.

D

DSI

An alternative name for the DfE Sign-in service.

DfE Manage

An admin console for DfE Sign-in for internal DfE staff only.

DfE Support

An admin console for DfE Sign-in for DSI BAU staff only.

E

End user

A basic permission level for users.

H

Home URL

The home page of the service users want to configure. It is usually the service landing page from DfE Sign-in.

I

IDAMS / Pirean

A legacy system which previously hosted services which now use DfE Sign-in.

ID-only services

An ID-only service uses DfE Sign-in for authentication only. These types of services do not have different levels of access for users.

Legal name

The registered legal name for an organisation.

Logout redirect URL

The logout redirect URL is the page that users want to redirect users to after they log out of the service. It is usually the service landing page from DfE Sign-in.

M

N

O

OIDC

A simple user identity and authentication layer on top of the OAuth 2.0 framework. DfE Sign-in is an OpenID Connect (OIDC) provider.

Organisation ID

P

Pirean / IDAMS

A legacy system which previously hosted services which now use DfE Sign-in.

Post password-reset URL

Where users want to redirect their users after they have reset their password. It is usually the DfE Sign-in home page.

Q

R

Roles

A role is a set of permissions that can be assigned to service users. They are also known as sub-services.

Redirect URL

The redirect URL is where users want to redirect users to after they have authenticated. DfE Sign-in will send the authentication response to this URL.

Response types

A response type is a value that determines the authentication flow. It is sent as part of an authentication request.

Refresh token

The refresh_token is a grant type. Grant types are the way that a service can get OAuth access tokens.

S

Sub-service

A sub-service is a set of permissions that can be assigned to service users. They are also known as roles.

Standard service

A standard service on DfE Sign-in allows you to add different levels of access for users, including roles, sub-services and policies.

T

Token endpoint authentication method

A token endpoint is where applications make a request to get OAuth access tokens. A token endpoint authentication method is the way your service authenticates to the DfE Sign-in token endpoint.

U

UK Provider Reference Number (UKPRN)

This is the unique identifier given to higher education providers.
