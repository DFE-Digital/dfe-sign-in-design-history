MFA - Disable MFA request journey

User journey for when an end user requests MFA to be disabled from an approver

From: DfE Sign-in Team

Published: 3rd October 2023

Last updated: 4th October 2023

Journey overview

As an approver one of the main responsibilities is to manage users at their organisation therefore the responsibility for approving or denying requests therefore when an end user requests to disable their MFA this responsibility are for approvers.

This journey shows the process of how an approver will respond to an end user who has requested that their MFA should be temporarily disabled because they cannot access their account due to the fact they cannot access their MFA method.

Users who request to disable their account, if approved, will have their MFA disabled for the day in order to give them access, this journey should only be used as a last resort for example if a user requires immediate access to complete an urgent action that same day.

Who can access this journey?

  • All approver users in DfE Sign-in
