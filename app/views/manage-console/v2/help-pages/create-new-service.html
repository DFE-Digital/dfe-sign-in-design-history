Create new service

As a developer in the DfE Sign-in team, to create a service on DfE manage console, you need to:

  1. Sign into your DfE Sign-in account.
  2. Once signed in, select ‘DfE Sign-in manage’ from your list of services.
  3. On the DfE manage console dashboard, click ‘Create new service’.
  4. Enter the relevant fields.
  5. Review details and click ‘Create new service’.

When you create a new service, you must provide the service’s:

  • type
  • name
  • description
  • home URL
  • post password-reset URL
  • client ID
  • redirect URL
  • logout redirect URL
  • response type
  • API secret

Depending on the response type selected, you may also need to provide:

  • refresh token
  • client secret
  • token endpoint authentication method

Service type

When creating a new service, you must choose the service type.

There are 2 types of service you can create:

  • standard service: services that allow you to add different levels of access for users, including roles, sub-services and policies
  • ID-only service: services that only use DfE Sign-in for authentication, meaning you cannot add roles, sub-services or policies

Standard service

A standard service on DfE Sign-in allows you to add different levels of access for users, including roles, sub-services and policies.

Some examples of where this would be needed include:

  • if different organisations need different information or functionality: a role could be created to give certain permissions to perform specific functionality
  • if a service needs internal admin users: a role could be created to give a user specific access to an admin page

Roles and sub-services are managed by policies. Policies are a group of conditions that must be met for a user to request a role. For example, a policy could state that users must be in a DfE organisation to request access to an admin role.

ID-only services

An ID-only service uses DfE Sign-in for authentication only. These types of services do not have different levels of access for users. This means you cannot add roles, sub-services or policies to an ID-only service.

Some examples of where this would be useful include:

  • a service based around a form: where all users need the same level of access and the service only needs information of a user and their chosen organisation
  • a service that needs the same level of access for all users: where all users have the same permissions to perform the same functionality

ID-only services cannot be seen in the DfE Sign-in service list.

Find out more about managing policies and managing roles.

Service name

When creating a new service, you must choose a name for the service.

A service name should:

  • be 500 characters or less (we recommend 65 characters or less)
  • be unique, clear and descriptive

Once the service has been created, you can change the service name without having to contact the DfE Sign-in team.

Service description

When creating a new service, you must include a short service description.

The service description will appear at the front end of the service, and should tell users:

  • what the service is for
  • what it can do
  • who can use it

To keep a service description understandable and relevant, it should be specific, informative and concise.

When writing a service description, you should:

  • be succinct, descriptions should be short (there is no character limit, but we recommend 500 characters or less)
  • use plain English
  • be descriptive
  • avoid jargon or acronyms that are not explained
  • front-load the description, with important information at the beginning
