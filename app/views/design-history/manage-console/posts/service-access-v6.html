About service access

  • Changed 'Manage policies to 'Manage Service access'

    • Policies are business rules located within service access that help control who approvers can assign roles to. The policy owner can grant specific roles to specific organisations.  We changed 'Manage policies' to 'Manage service access' as an umbrella term above 'Policies' so that it would be easier for users to understand.

Create policy

  • Limited the organisaton policy

    • We limited the organisation policy to just the organisation and role fields, where users can select multiple fields if necessary.  Users have the ability to filter through to find their organisations, if necessary.

  • Limited the user policy

    • We limited the user policy to just the email and role fields, where users can select multiple fields if necessary.

Conditions

  • Changed the language conditions to rules

    • Users found the 'Conditions' language confusing, so we changed it to 'Rules' as these are business rules that make up the policy.

Existing policies

  • Policies already within the system

    • Policies within the existing system will still be displayed, but we are not including 'Change policy' or 'Remove policy' for these existing policies. Instead, we have a warning message telling users to create a 'Service Now' request.
