Transfer prototype from Invision Studio to Gov prototype kit

  • Transferred the entire Manage Console prototype from Invision Studio to the Gov prototype kit

    • Invision Studio as of 2023 is no longer supported or available and has been sunset as a prototyping product. The entire Manage Console prototype existed in Invision Studio therefore it was necessary to transfer all existing pages to the prototype kit to ensure it is supported moving forward.

Organisation & user management filters

  • Created filters for Organisation search & User management sections of Manage Console

    • Users will now be able to filter organisations by Status & Organisation category. In user management users will be able to filter users by User type, Organisation type, Last login & Account status.

  • Added an apply filter button to the filters

    • Added an apply filters button to the filters, this feature was required as the developers required a button to prevent the page from refreshing after each filter has been selected. Users can now choose multiple filters without the page refreshing and can refresh the page using apply filters to show the filtered table.

Service configuration

  • Create section headings

    • Created section headings for: service details, OpenID Connect & API as well as providing descriptions for each section.

  • Moved input field labels above the input field

    • The label of each input field has been moved above each input field to follow GDS guidance and patterns and to make the page more accessible.

  • Added hint text to each input field

    • In user research users expressed that they were unsure of what some of the input fields and labels meant, hint text has been added underneath the label to help users understand what each field is and does.

  • Extended the length of input fields

    • Due to the character length of some of the inputs which users were required to enter it was required to extend the input field lengths to follow GDS guidance and guidelines.

  • Added the ability to remove fields

    • Added the ability for users to remove input fields where required for example Redirect URL and Logout Redirect URL as users can enter multiple URL's and required the ability to remove URL's.

  • Added an Add URL button

    • Added an button to replace the Add URL link to follow GDS guidance and to ensure it was clear to users where and how they could add URL's.

  • Updated the Response types & removed Grant types

    • To follow best practice in line with OIDC guidance we updated the response types to show the correct values & removed grant types as users should not have the ability to select this and should instead be automatically selected for them.

  • Added a warning message related to Response types

    • Added a warning message to the Response types category as when a user makes a selection a flow type is automatically selected for them. The warning message is displayed to let the users know which flow has been selected.

  • Added a refresh token field

    • Added a refresh token field for users to select, this was previously found under Grant types however required a section of it's own as it is classed as a separate field in OIDC guidance.

  • Moved the client secret input field

    • Moved the input field for client secret underneath the repsonse types as the client secret input field is only required depending on what selection the user has made in response types.

  • Aligned 'show' buttons with the input field

    • Aligned the show buttons linked to the input field inline with the input field to ensure it is clear to users that the button relates to the field it it alongside rather than underneath the field.

  • Updated the selections users can make under Token endpoint authentication method

    • Updated the options under token endpoint authentication method to align with the OIDc guidance and requirements.

  • Made some fields conditional to appear depending on what options the user has selected

    • Under OIDC guidance Refresh token, client secret and toekn endpoint authentication method should only be selected or made available depending on which refresh token options the user has selected therefore certain fields will not appear until a selection is made to prevent users selecting the wrong setup.

  • Created a review service configuration page

    • Created a review service configuration page to give users the opportunity to review the changes they have made in order prevent users making mistakes potentially causing service outages.

Service onboarding

  • Created service onboarding journey

    • Created a new journey which previously didn't exist to allow users to onboard a service to DfE Sign-in.

  • Updated the journey to replicate the service configuration pages

    • Updated the journey to replicate the service configuration page to ensure consistency for users between the two elements.

Manage policies

  • Remove tabs when a user is viewing policy information

    • Removed tabs and displayed the roles and consitions tables on the same page, allowing users to see both tables at the same time without clicking enabling comparison between the two table easier.

  • Added a description to the manage policies page

    • Added a description of what policies are and what users can do to ensure users understand what they can do.

  • Added links to help pages related to manage policies

    • Added links to manage policies help pages under related actions to ensure users can quickly access help when they need it.

  • Added additional information to the policy pages

    • Added additional information to the policies page such as policy status and policy type to ensure users have the information they require.

  • Added the ability for users to view individual roles related to a policy

    • Users can now click the name of a role and be taken to a role page which previously didn't exist, this enables users to view all policies related to a specific role.

  • Added the ability to create a new policy

    • Added the ability to create a new policy, enabling them to update their policies to ensure users have the right access to the services they require.

  • Added the ability to change and edit existing policies

    • Added the ability to change and edit exisiting policies ensuring users can maintain a policy after they have created it to ensure that users can access the right services when they require it and prevent users accessing services which they don't require.

  • Added the ability to remove and delete a policy

    • Added the ability to remove and delete a policy from a service, allowing users to keep their policies up to date and remove older policies which are no longer required.

Help pages

  • Created the help section for help articles

    • Created the help section for where users can go to get help and answers to their questions in the form of help articles.

  • Added About DfE Manage console help page

    • Added a help page to the help section called about DfE Manage Console.

  • Added manage users help page

    • Added a help page to the help section called Manage users.

  • Added Search organisations help page

    • Added a help page to the help section called Search organisations.

  • Added Edit service configuration help page

    • Added a help page to the help section called Edit service configuration.

  • Added Create new service help page

    • Added a help page to the help section called Create new service.

  • Updated most popular help links

    • Updated the most popular help links to include the new help pages.
